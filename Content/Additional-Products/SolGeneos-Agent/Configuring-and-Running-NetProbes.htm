Installing, Upgrading & Running Netprobes

This section describes how to install and run Netprobe on an event broker. If you want to use a Netprobe deployed on a server external to the event broker, it must be installed and configured according to ITRS guidelines.

Prerequisites

  • ITRS Netprobe archive: 64-bit version for Solace PubSub+ software event broker 8.2 or higher.
  • Additionally, the same prerequisites apply as stated in Installing/Upgrading SolGeneos Agent.

Installation & Upgrade

To install or upgrade Netprobe, do the following:

  1. Ensure any old Netprobe installation is stopped and removed. See the Uninstall Netprobe steps below.
  2. Create a new directory (if required) and change directory to /usr/sw/geneos.
  3. Extract the ITRS Netprobe archive to /usr/sw/geneos.
  4. Copy all contents of the /usr/sw/solgeneos/loads/solgeneos_<version>/templates/netprobe directory to /usr/sw/geneos.

Configuration

To configure and run Netprobe, do the following:

The Netprobe-install.txt file, copied locally from the templates directory in step 4 above, provides step-by-step instructions.

  1. It is recommended to install Netprobe as a service so that it auto-starts at boot time. Alternatively, you can manually start a Netprobe instance.
  2. If required, the default listening port 7036 can be changed.
  3. If required, multiple Netprobe services can be configured, each listening to different ports.

    Once a Netprobe is started through the script, it must be configured on the ITRS Geneos Gateway.

Uninstall

To uninstall Netprobe, do the following:

  1. Ensure any old Netprobe service is stopped. Consult the Netprobe-install.txt file for any details.

    Any Netprobe instances running on the event broker prior to the first installation of the Solace Geneos Agentwill not stop when the script is installed. In addition, they will not restart if the Netprobe processes crash or if the event broker restarts.

  2. Remove all contents of the /usr/sw/geneos directory.
