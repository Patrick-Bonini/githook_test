Solace Geneos SEMP Service Down Alert

You can add a rule to see an alert when the SEMP service is down. By adding the rule, the data view under Solace Sampler turns to red, and the stale data is greyed out. If the rule is not created and the SEMP service goes down, stale data will be greyed out, but the data view under Solace Sampler will not change.

The rule can be manually added as follows:

  1. In the Gateway Setup Editor, expand the rules folder.
  2. Select the Solace Rule group.
  3. Click on New Rule.
  4. Give the rule a name of your choice.
  5. Add the following target to the rule:

    /geneos/gateway[(@name="system-test")]/directory/probe/managedEntity/sampler[(@name="SolaceSampler")]//rows/row/cell[(@column="/FAIL")]

    Rule target may vary depending on the environment. When setting the rule target, do not specify a dataview. This ensures that the rule applies to all SolGeneos monitors.

  6. Give the rule a block as follows: severity critical.
  7. Save configuration.
