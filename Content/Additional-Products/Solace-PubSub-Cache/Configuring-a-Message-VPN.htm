Configuring Message VPNs for Caching

The following considerations should be made for a Message VPN that will contain the Distributed Cache:

  • If the Global Caching feature is to be used, the subscription export policy must be enabled (for information, refer to Considerations for Implementing Global Caching). By default, this policy is not enabled.
  • If the new topic advertisements feature is to be used, the publishing of Message VPN events must be enabled (for information, refer to Publishing Message VPN-Level Events. By default, this policy is not enabled.
  • For releases before 10.0.1, Distributed Cache Management services must be enabled for the Message VPN (for information, refer to Enabling Distributed Caches). Distributed Cache Management services allows the event broker that the Message VPN is hosted on to manage the configuration of PubSub+ Cache Instances and to synchronize them.

    The event broker with a Message VPN that has Distributed Cache Management services enabled acts as the Designated Router for that Message VPN. By default, when a Message VPN is created, Distributed Cache Management services are enabled.

    In addition for releases prior to 10.0.1, Distributed Cache Management services must be enabled for a Message VPN, otherwise service will be unavailable. However, if a Message VPN of the same name is hosted on multiple event brokers that are connected in networked deployment, Distributed Cache Management services must only be enabled on one of those Messages VPNs. For more information, see Redundancy and Network Deployments.
