Configuring Message Replay

Message replay allows event brokers to resend messages to new or existing clients that request them, hours or sometimes days after those messages were first received by the event broker.

When you enable message replay, event brokers store persistent messages in a replay log. These messages are kept until the log is full, after which the oldest messages are removed to make space for new messages.

Message replay can be performed for a non-partitioned queue or a topic endpoint. When you initiate a replay, you can request all messages in the replay log, all messages following a specified message ID, or all messages starting from a requested replay start time. The event broker delivers any messages held in the replay log that match any subscription on that queue or topic endpoint.
