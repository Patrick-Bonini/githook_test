Creating a Client Username

Client usernames are used for client applications to authenticate and are associated with a client profile and ACL profile to control the permissions of the connecting client application. Client usernames are specific to a Message VPN. They are used by client applications to authenticate with the event broker, and may be used to make multiple client connections. This lets applications horizontally scale without additional configuration on the event broker. Each client username is associated with a client profile and an ACL profile. These profiles control properties and permissions of the connected client application.

For instructions to create client usernames in , see Creating Client Usernames.

To create a client username for an or , use the following steps after logging into with at least Message VPN scope and read-write access level:

  1. In , on the Message VPN page, select Access Control from the navigation bar.
  2. On the Access Control page, select the Client Usernames tab.
  3. In the top-right corner, click the +Client Username button.
  4. In the Create Username dialog, enter a name for your client username and then click Create.
  5. On the Edit Client Username Settings page.
    1. Toggle on Enable.
    2. Click Change Password and enter a new password in the New Password and Confirm Password fields.
    3. In the Client Profile list, select the profile to use with the client username.
  6. Click Apply.

You've now created a client username that can be used by client applications to access messaging services on the event broker.
