Setting the Timezone (Machine Images)

Distribution of PubSub+ packaged as a virtual machine broker image will cease as of release 10.8.1 (June 2024). For more details, see the Deprecated Features list.

Solace recommends that you transition to an alternate method of deploying PubSub+ prior to June 2024.  Containers offer a flexible way to deploy PubSub+ in a number of environments, virtual machines,  and other container platforms. For information about:

  • Setting up containers, see Setting Up Container Images.

  • Deploying PubSub+ container images in production environments, see Deploying PubSub+ Software Event Brokers in a Production Environment.

To configure the timezone for the host on software event broker machine images you can use the timedatectl set-timezone <TIMEZONE> utility. After setting the timezone on the host, it is recommended to reboot. The timezone between the host and the container can be synchronized when the container is restarted. You can check the configured timezone using the show clock Solace CLI command. To see the list of available timezones, use the timedatectl list-timezones command.

To set the timezone for machine images using cloud-init, refer to Timezone Initialization.

In addition, The CA machine image includes VMware Tools and Chronyd to support time synchronization.

To verify that VMware Tools time synchronization is enabled enter the following command:

[sysadmin@solace ~]$ vmware-toolbox-cmd timesync status
Enabled

If you are not using VMware Tools for time synchronization, you can configure and enable Chronyd to synchronize via NTP. For more information, see Configuring Network Time in the Oracle Linux 9 documentation.
