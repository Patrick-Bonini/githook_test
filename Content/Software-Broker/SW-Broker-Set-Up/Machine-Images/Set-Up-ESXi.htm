Before You Begin

It's assumed that you have VMware ESXi installed on your host system.

VMware Tools Support

The PubSub+ supports VMware Tools when running in a compatible hypervisor environment (ESXi or VMware Workstation Player) as follows:

  • It supports VMware Tools functionality including Managed Power Settings, Enhanced Networking, Network Status, and Heartbeat.
  • It doesn't support memory ballooning and time synchronization.

System Requirements

The software event broker requires storage with medium to high bandwidth/IOPS and low latency. The virtual machine image uses settings for the widest compatibility; these settings are not tuned for performance. For information about getting the best storage performance for your system, consult the VMWare documentation.

High Availability Considerations

Step 1: Get a Software Event Broker

The first step is to download an edition of PubSub+ . For VMware ESXi, the event broker is distributed as an Open Virtualization Archive (.ova) file.

  1. Go to the Solace PubSub+ Event Broker downloads page.
  2. Scroll to the OVA section. Fill in your details, and then select the required edition of the software event broker.
  3. Fill in your details, and then click Get The Downloads.
  4. Click Download under the PubSub+ edition.

A file called solace-pubsub-<edition>-<version>.ova is downloaded.

Step 2: Import the Software Event Broker

To create a new VM for the event broker, you must import the OVA file into the VMware ESXi host. Once it is imported, you can start the event broker instance.

To do this, follow these steps:

  1. Launch VMware ESXi.
  2. On the main dashboard, select Create/Register VM to create a new virtual machine .

    The New Virtual Machine Wizard opens.

  3. From the creation type menu, select to deploy your VM from an OVF or OVA file. Then, click Next.

  4. Provide a name for your event broker instance and import the OVA file that you downloaded (by clicking the blue box or dragging the file into the box). Then, click Next.

  5. Select a location for the event broker storage. Then, click Next.

  6. Select the format of the virtual disk that the event broker will use (thick provisioning is recommended). Then, click Next.

  7. Review the settings and then click Finish.

    The event broker VM starts the installation process.

  8. Open the console.

    Right-click the event broker VM and select Open browser console from the Console tab.

  9.  Login to the event broker.
