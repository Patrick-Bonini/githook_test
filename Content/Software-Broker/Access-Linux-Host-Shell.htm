Accessing the Linux Host Shell (Machine Images)

Distribution of PubSub+ packaged as a virtual machine broker image will cease as of release 10.8.1 (June 2024). For more details, see the Deprecated Features list.

Solace recommends that you transition to an alternate method of deploying PubSub+ prior to June 2024.  Containers offer a flexible way to deploy PubSub+ in a number of environments, virtual machines,  and other container platforms. For information about:

  • Setting up containers, see Setting Up Container Images.

  • Deploying PubSub+ container images in production environments, see Deploying PubSub+ Software Event Brokers in a Production Environment.

The Linux host shell is the shell for the guest operating system that contains the Solace PubSub+ software event broker application container. This environment can be accessed through the serial console or ssh to port 22. The intended use of the Linux host shell is for basic system administration activities such as setting passwords, managing software event broker system services, and configuring storage for the message spool. The solacectl helper utility, which automates some common tasks, runs in this environment (for more information, see Using the Solacectl Utility). The Solace CLI and application container shell environments can also be accessed from the Linux host shell through the solacectl application.

The Linux host shell is accessible via:

  • serial console (hypervisor environments)
  • ssh to port 22 (cloud environments) with user sysadmin
