Changing the VRID for Active/Standby Appliance HA Pairs

The procedure shown in this section allows you to change the VRID of an active/standby HA appliance pair.

The procedure isn't applicable to software event brokers.

  1. Enter the shutdown Redundancy CONFIG command on both the primary and backup event brokers to stop redundancy.
  2. Enter the shutdown Message Spool CONFIG command on both the primary and backup event brokers to stop Guaranteed Messaging and message spooling.
  3. Enter the vrrp-vrid Redundancy CONFIG command on the primary event broker to reconfigure the VRID.

    If you change a VRID IP address on an event broker while it has active subscriptions, those subscriptions remain in the SMRP database of all other event brokers in the network. You must then remove these subscriptions manually, event broker by event broker, using the delete-remote-router Admin EXEC command.

  4. Enter the vrrp-vrid Redundancy CONFIG command on the backup event broker to reconfigure event broker redundancy on it to specify the new VRID of its mate.
  5. Enter the assert-disk-ownership Message Spool Admin EXEC command on the primary event broker to assert ownership for the new VRID.
  6. Enter the no shutdown Redundancy CONFIG command on both the primary and backup event brokers to start redundancy.
  7. Enter the no shutdown Message Spool CONFIG command on the primary event broker to start Guaranteed Messaging and message spooling.
  8. Enter the no shutdown backup Message Spool CONFIG command on the backup event broker to start Guaranteed Messaging and message spooling.
