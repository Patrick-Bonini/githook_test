Disconnecting Bridge Connections

You can disconnect a bridge from either the local or remote Message VPN, which causes the TCP connection for the bridge to be disconnected. The disconnected bridge immediately attempts to reconnect, starting at the first remote Message VPN in its configured list.

To disconnect a Message VPN bridge and its clients from the Solace PubSub+ appliance, do the following:

  1. Enter the following command:

    solace# admin
    solace(admin)# bridge <bridge-name-pattern> message-vpn <vpn-name> [primary | backup]

    Where:

    <bridge-name-pattern> is the Message VPN bridge name. This can be its full name or part of its name when wildcard characters are used.

    <vpn-name> is the name of an existing Message VPN that the bridge belongs to. This can be its full name or part of its name when wildcard characters are used.

    primary indicates only bridges of the primary virtual router

    backup indicates only bridges of the backup virtual router

  2. Enter the following command:

    solace(admin/bridge)# disconnect
