Only the supports auto-instrumentation, all other s require manual instrumentation.

  • You must upgrade your OpenTelemetry Collectors to the minimum version listed above before you upgrade the event brokers that the Collectors are receiving OpenTelemetry data from.
  • Version 10.2.0 of the event broker includes the initial implementation of the distributed tracing feature and requires versions of the OpenTelemetry Collector that include the .
  • Version 10.2.1 of the event broker is the initial implementation of context propagation and is the minimum event broker release required to work with context propagation on any .

    To use context propagation, you must have the minimum required versions of the OpenTelemetry Collector as well as the minimum API release the Collector requires.

  • Version 10.4.0 of the event broker is the initial implementation of partitioned queues.

    If you want your trace messages to include the associated queue partition, you must have the minimum required versions of the OpenTelemetry Collector as well as the minimum API release the Collector requires.

  • Version 10.4.1 of the event broker adds support for an egress span containing information on the transmission to and response from the consumer, including the outcome (ACK/NACK).
