Enabling Advanced Monitoring

Advanced Monitoring lets you access more advanced monitoring functionality and details directly within Datadog (the central monitoring service). Advanced Monitoring is a feature of and is enabled by an account administrator. Datadog collects monitoring data from s no matter the deployment. Advanced Monitoring provides you with access to Datadog and its rich ecosystem of dashboards that many Operations, Integration, and Application Support teams use to analyze monitoring data. This monitoring data is processed by Datadog to build meaningful historical and real-time metrics that you can use to analyze the health and performance of the s that you've deployed.

You can configure access to the monitoring data in Datadog with a subscription to . Access to provides you with valuable information to manage and get insights into your s. With direct access to Datadog, you can customize and build your own dashboards to higher levels of granularity that may better suite your use case and requirements.

Direct access to Datadog requires that you have an understanding of how to work with Datadog dashboards. Not all users in your account may require this access. When giving users in advanced event mesh for SAP Integration Suite the role, a corresponding Datadog account is created for that user.

Considerations for Providing Direct Access to Datadog

Here are some considerations for providing access to the monitors and metrics within Datadog directly.

  • If you're familiar with Datadog, as part of your subscription, sets up a sub-organization for your account so that you can see information that's related to your account. If your subscription to ends, access to that Datadog account ends as well.

  • If your organization has the controlled availability feature to configure single sign-on for the Datadog account, you can map user permissions to existing roles. For more information see Single Sign-On for [%=Product-Names.insights_long%] Datadog Account as Controlled Availability.

  • With your subscription, you have access to your own sub-organization containing the metrics, monitors, and dashboards for your account. If you belong to multiple accounts, you may need to select the proper sub-organization when you log in to Datadog directly. Any user that has the role of is granted the necessary permissions to manage their monitors and dashboards, including:
    • (read, write, and public share)
    • (read, write, and manage downtime)
      • The write permission only provides the ability to edit cloned monitors. The default Insights monitors are locked and cannot be edited.
    • Logs read Index Data
    • Logs read data
    • Trace data if you have a subscription to Distributed Tracing.

    Users with the role of are granted permissions to view dashboards and monitors. They cannot manage or change monitors and dashboards. They can also read trace data if you have a subscription to Distributed Tracing.

Configuring an Existing User to Access Datadog

You can configure an existing user's (or new user's) access to Datadog provided you have the Administrator role in advance event mesh. To give access to Datadog, you assign the or role, which triggers a separate invitation to the user from Datadog. For more information about configuring roles when you add a user, see Adding a New User.

The following steps are to provide an existing user access to a Datadog account.

  1. Log in to the if you haven't already.
  2. On the navigation bar, select User & Accounts and then Account Details.
  3. On the Account Details page, select the User Management.
  4. Click Actions beside the name of the email of the user you'd like to enable direct access to Datadog, and select Edit.
  5. On the Edit User Roles dialog, select either or role in the Datadog Access field and then click Save.

After these steps, an invitation is sent to the user that you assigned the role or to.

Removing Datadog Access for a User

You can remove Datadog access from an existing user provided you have the Administrator role in advance event mesh. When access is removed for a user, the corresponding Datadog account for that user is disabled.

The following steps are to remove access to Datadog from a user.

  1. Log in to the if you haven't already.
  2. On navigation bar, select User & Accounts and then Account Details.
  3. On the Account Details page, select the User Management.
  4. Click Actions beside the user whose access to Datadog you'd like to remove, and then select Edit.
  5. On the Edit User Roles dialog, remove the , , or role from the Datadog Access field and click Save.

After completing these steps, the user's Datadog account is disabled.

Resending an Invitation Email from Datadog

Invitation Emails for Datadog accounts expire after a time if the user doesn't accept the invitation. As an administrator of advance event mesh, you can resend the Datadog invitation to the user if they did not activate their account in time. To do this, perform the following steps:

  1. Log in to the if you haven't already.
  2. On navigation bar, select User & Accounts and then Account Details.
  3. On the Account Details page, select the User Management tab.
  4. Identify the user for whom you want resend the invitation. Optionally, you can type in the Search Emails field to filter the user accounts that are visible.
  5. Beside the user for whom you want to resend an invite, click the Actions and select Re-send Datadog Invite. Note that this option is available only if the user hasn't accepted their invitation to Datadog.

Have users check their spam folders or junk mail to ensure that the invitation email has not been filtered.

The user should receive an invite shortly. For more information about accepting the invitation, see Accepting a Datadog Invitation.

Accessing Datadog from

After you've activated your Datadog account, you can directly access Datadog from your account in advance event mesh. This is particularly useful if you have multiple accounts on advance event mesh.

To do this, perform the following steps.

  1. Log in to the if you haven't already.
  2. On the navigation bar, select and then click on a service that you want to see.

  4. Click Open Advanced Insights in Datadog for a service in your account to go directly to the Datadog dashboard for your service.

A separate browser window or tab appears to access your Datadog account.

Sharing a Dashboard Using a URL

If you have the or role, you can generate a shared URL for a dashboard. This gives you the ability to share a dashboard with people who do not usually have access to the dashboards that are provided with . The URLs shared this way are managed by you, and can be limited in who they are shared with, and the length of time that the URL is accessible. This allows you to share the valuable information in Datadog without needing to change your permissions.

Any dashboard that is being shared is indicated as such in the list of Dashboards in Datadog with a SHARED icon, as shown in the image below:

To share a using a URL

  1. In Datadog go to Dashboards > Dashboard List to access the list of monitors.
  2. Select a dashboard you want to generate a shared URL for by clicking on it in the list.
  3. Click on the gear icon in the top right corner and select Generate shared URL from the menu that opens.
  4. The Sharing window opens.

  5. From this window you can select who to share the URL with, and what time and variable settings you want to configure if any
    • Use first field to specify who to share the URL with:
      • Only specific people - You can use the provided field to specify who to share the link with by individual or group email address.
      • Anyone with a link (public) - The URL The link will display in the field below. You can copy this link and share it with anyone you want to provide with access to the dashboard.
    • Clicking on the Time & Variable Settings opens a screen where you can configure various elements that will be displayed on the shared dashboard, including:
      • The time visible on the shared board, and whether users can change the time frame they are viewing.
      • Use the Default and Available value fields to select specific services to from their associated menus.
  6. When you are ready to share the URL click Done.

    7. If you specified to share dashboard with specific people, they will receive an email from Datadog with the URL. The link in the email is good for one use, and will expire within one hour of being sent.

    If you selected to share with anyone, you will need to provide the URL to whomever you want to share the dashboard with. When the URL is shared with anyone, you can copy it from the menu or the Sharing window and email it to a person. Sending a URL designed for specific people will not work in this manner.

    When sharing of a dashboard is enabled, the menu accessed by clicking the icon will acquire a Sharing section. This section will indicate the shared status. You can also copy the URL from this section, and click Configure sharing to access the Sharing window.

Revoking a shared URL

You can revoke the URL for a shared dashboard at any time.

  1. In Datadog go to Dashboards > Dashboard List to access the list of monitors.
  2. Select a dashboard you want to stop sharing by clicking it in the list.  Note that any dashboard that is currently being shared by URL will have a SHARED icon

  3. Click on the gear icon in the top right corner and select Configure sharing from the menu that opens to access the Sharing window.
  4. Click Revoke URL and then Revoke Access on the confirmation window that opens.

Revoking the URL immediately stops sharing the associated . Anyone accessing or attempting to access the revoked URL receives a 404 page from Datadog.
