You can configure management access to s to use the same single sign-on (SSO) via OpenID Connect (OIDC) as the . Using SSO allows users to authenticate with the same credentials they use to access the . Authorization for management access is handled with claims configured for your identity provider (IdP).

SSO configures management access through to s. Access to the messages and events (or machine-to-machine authentication) uses a different authentication mechanism (for example, basic authentication, LDAP, client certification, or OAuth provider) that is configured separately for client applications. Considerations for Using SSO

Before you configure SSO using OIDC, your account must meet the following requirements:

  • You must have an Enterprise Account for . Trial accounts do not support SSO configuration via OpenID Connect (OIDC).
  • The Administrator role must be assigned to the user in the account to enable SSO. Users with role cannot enable SSO on an .
  • The roles in your Enterprise Account must be mapped appropriately to the roles in your identity provider:
    • the Administrator and roles give edit permissions to the Message VPN on the
    • the role give read access to Message VPN on the
  • If you have private and public endpoints configured for the , the redirect URIs need to be configured on the IdP. SSO via OIDC must be enabled on the account.
  • Group management must be set up on the account.
  • Just-in-time provisioning must be configured on the account.

After the SSO configuration is enabled on the , users have management access (for example ) via SSO to the . You can also check the Management Access on the on Status page for the service. For more information, see Detailed Information About [%=Product-Names.broker_cloud_short_title%]s.

SSO access is only for management access for users. Access to the messages and events (or machine-to-machine authentication) use a different authentication mechanism (for example, basic authentication, LDAP, client certification, or OAuth provider) that is configured separately for client applications. For more information, see Configuring Authentication.

You can also configure SSO using the REST API for . For more information, see Managing Single Sign-On for [%=Product-Names.broker_cloud_short_title%]s using OpenID Connect with the [%=Product-Names.cloud_product_titlecase%] REST API.
