offers two release types for : Production and Preview. Both are generally available releases available for Kubernetes-based deployments.

Production releases are generally available software intended for production use and offer the following:

  • High-severity and security fixes are available within 30 days of availability of an industry-accepted remedy.

  • Solace releases qualified security errata and selected urgent bug fixes as they become available. The qualified security errata includes critical and high Common Vulnerability and Exposures (CVEs), and critical and important Red Hat Security Advisories (RHSAs).

Production releases in differ from in the following aspects:

  • Whether the upgrade of existing s to that release is supported; when upgrade is supported, you can only upgrade when the deployment type is the same as your deployment type.

Preview releases are intended for new feature introduction and integration. Preview releases provide a subset of the features to give you a preview and an indication of what is available in the next Production release. They have the following limitations when compared to Production releases:

  • offers only technical support for Preview releases (six months after the release date of the ).

  • No maintenance loads are provided. any addresses critical bugs or security issues identified in a Preview release in the next Production release.

  • You can only create new with Preview releases. Upgrades to the Preview release from existing are not possible or supported.

  • The upgrade path for a Preview release is to upgrade to the subsequent Production release.

Support Phases in

Each Production release has a full support phase and a technical support phase. The technical support phase begins after the full support ends. The technical support phase provides adequate time to upgrade existing s to the most recent Production release (that supports upgrades).

Because Preview releases are intended for feature introduction and integration rather than production purposes, they offer only a technical support phase. If you want to upgrade a Preview release, you can upgrade only to a newer Production release.

The full support phase for Production releases begins on the date when the is released and not the date when the version is made available in . For example, version 10.3.1 (Production) of the was released in March 2023, and became available in in April 2023. Full support for this version ends in March of 2024 (one year from the release) and technical support ends in September of 2024 (18 months after its original release).

Technical support for Preview releases begins on the date when the is released and not the date when the version is made available in . For example, if a Preview version of the is released in September 2023 and becomes available in in October 2023. Technical support for this version ends in March of 2024 (six months from the release).

Starting in 10.4.0, the underlying versions are released eight times a year. Full support for the version ends one year from when Solace released the version. For the dates, see Software Event Broker Versions and Support in [%=Product-Names.cloud_product_short%].

Technical support for Preview releases begins on the date when the is released by Solace and not the date when the version is made available in . For example, if a Preview version of the is released in September 2023 and becomes available in in October 2023. Technical support for this version ends in March of 2024 (six months from the release).
