Configuring User Access to

The level of access you have to the features and resources in depends on the role you're assigned in and the permissions granted to that role. Roles can be assigned directly to users or, if your organization has SSO enabled, to user groups. For more information about assigning user roles, see Managing Users, Groups, Roles, and Permissions.

s
users with the role have limited access and viewing capabilities. They can't open an application domain or modeled event mesh unless they are granted access by someone with Manager level access. The following access levels can be granted to users and user groups with the role:
s with Viewer access to an application domain can perform these functions:
  • view the objects in the application domain
  • download the AsyncAPI document for applications and event APIs in the application domain
s with Editor access to an application domain perform the same functions as users with Viewer access. They can also create, update, and delete objects in the application domain.
s with Manager access to an application domain can perform the same functions as users with Editor access. They can also perform these functions:
  • change an object's version state
  • modify the application domain information, including name, description, and topic domains
  • add and delete custom attributes in the application domain
  • give users with the role access to the application domain
  • give users with the role the event access approver role for the application domain
Event Access Approver
s with approver access to an application domain can approve and decline event access requests. They can also perform the same functions as users with Application Domain Viewer access.
s with Viewer access to an environment can view the objects in modeled event meshes that are part of the environment in . To open objects in the modeled event mesh, users must also have access for the application domain that contains the objects.
s with Editor access to an environment can perform the same functions as users with Viewer access. They can also perform these functions:
  • add and remove objects from the modeled event mesh if they have at least Editor level access to the application domain that contains the objects
  • rearrange the graph layout
s
users with the role have full access to all Event Portal capabilities. They also have Manager level access to all application domains.
Administrators
users with the Administrator role have full access to all capabilities. Administrators can assign users any role in . In , Administrators have the same access as s.
