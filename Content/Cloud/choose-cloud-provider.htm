Choosing the Right Cloud Provider

Your choice of cloud provider should be based on the requirements of your event-driven architecture, and the requirements of the client applications connecting to your s. Some may only require a public cloud provider while others may require a provider with virtual private cloud or virtual network (VPC/VNet) capabilities. To accommodate both situations, offers both public and private cloud provider choices.

If a public cloud meets your requirements, provides s. The cloud providers available for s include Amazon Web Services (AWS), Google Cloud Platform (GCP), and Microsoft Azure. s are ideal for client applications that connect to s using the public Internet. s also require the least amount of setup, allowing you to quickly start using .

If you require a private connectivity, provides Private Clouds. Typically, you use private clouds when the between your s and client applications requires private IP addresses rather than using the public Internet, or when you require specific deployment options (such as on premises infrastructure or other cloud providers). provides two choices for Private Clouds:

  • s—-controlled VPC/VNets dedicated to your organization on public cloud providers, including AWS, GCP, and Azure. Various networking options are available.

  • s—Infrastructure you control, including on-premises or cloud-based Kubernetes clusters. Supported Kubernetes providers include: OpenShift, Rancher (RKE1), Amazon (EKS), Azure (AKS, ARO), Google (GKE), Alibaba (ACK), Huawei (CCE), and more.
