PingOne with

You can integrate PingOne with to enable single sign-on (SSO) and multi-factor authentication (MFA) for your enterprise accounts.

After you have configured PingOne, your organization will receive a customized URL to log in to . If your users have already signed in to their PingOne account, they will automatically be signed in to . Otherwise, they will be directed to your organization's PingOne sign-in page to enter their details. Your organization's PingOne settings determine whether multi-factor authentication (MFA) is used.

If a user belongs to multiple accounts that are associated with your organization's PingOne ID, they can switch between those accounts without logging in to the accounts separately.

Register as an Application in PingOne for Enterprise

When you set up as an application in PingOne for Enterprise, a Client ID, Client Secret, and Discovery URL is generated that will be used to bind your account to PingOne. These information allow your organization's users to be authenticated by PingOne.

To register as an application in PingOne for Enterprise, perform the following steps:

  1. Under Applications > My Applications, select OIDC, then click Add Application.

  2. On the Add OIDC Application dialog, do the following:
    1. Select Advanced Configuration and click Next.

    2. Provide details about your application as shown in example below and click Next.

    3. Under Authorization Settings dialog, add a Client ID and Client Secrets.

    4. In the SSO Flow and Authentication Settings dialog:
      1. Use the URLs provided by team in Start SSO URL field (example, customer-sso.solace.cloud) and Redirect URIs field (example, customer-sso.solace.cloud/sso/login). Refer to Request a Preferred Subdomain to learn how to get the URL.
      2. Click Next.

    5. Leave the default user profile attribute settings as it is and click Next.

    6. On the Connect Scopes dialog, select the OpenID Email and click Next.

    7. On the Attribute Mapping dialog, ensure that the value of the email attribute matches the email used in PubSub+ Cloud, and click Next.

    8. Select the groups to grant access to , and click Next to complete the setup.

  3. Make a note of the following information, which you will need when configuring your SSO in

    • Client ID
    • Client Secret

    • Discovery URL/Endpoint

Register as an Application in PingOne for Customers

When you set up as an application in PingOne for Customer, a Client ID, Client Secret, and Discovery URL is generated that will be used to bind your account to PingOne. These information allow your organization's users to be authenticated by PingOne.

To register as an application in PingOne for Customers, do the following:

  1. Select Connections > Add Application.

  2. Click on Advanced Configuration and select OIDC.

  3. Write the Application Name, and a Description, and click Next.

  4. In the Configure > Redirect URLs field, enter the redirect URL given to you by the PubSub+ Cloud team, then Save and Continue. Refer to Request a Preferred Subdomain to learn how to get the URL.

  5. Add the OpenID email to the Scope Grants and click Save and Continue.

  6. Use the default OIDC attributes as it is, and click Save and Continue.

  7. On the newly created Applications page, click the pencil icon to edit.

  8. Under Applications > Configuration, do the following:
    1. Select the following options:
      • RESPONSE TYPE: Code
      • GRANT TYPE: Authorization Code
      • TOKEN ENDPOINT AUTHENTICATION METHOD: Client Secret Basic

    2. Take note of the following attributes which will be required for your SSO configuration in Cloud
      • CLIENT ID
      • CLIENT SECRET
      • OIDC DISCOVERY ENDPOINT
    3. Click Save.
  9. Go back to the Applications page and enable user access to the application.

  10. Make a note of the following information, which you will need when configuring your SSO in

    • Client ID
    • Client Secret

    • Discovery URL/Endpoint
