Configuring an OpenID Connect Identity Provider

This topic provides basic instructions to configure an OpenID Connect (OIDC)-based identity provider. Note that the exact process may differ depending on the service provider. If your identity provider has specific documentation for configuring OIDC, recommends following that procedure.

After you configure your identity provider is configured, you use information from the provider to set up SSO in PubSub+ Cloud.

You can also refer to the documentation from these identity providers:

  • Microsoft Entra ID
  • Okta
  • PingOne
  • Auth0

Before You Begin

  • Verify that your identity provider supports the following:
    • Response Type: Code
    • Grant Type: Authorization Code
    • Grants/Claims: Email
    • Token endpoint authentication: Client Secret
  • Request a subdomain for your account. Your identity provider uses this subdomain to accept authentication requests

Configure an OIDC Identity Provider

To configure an OIDC-based identity provider, perform these steps:

  1. Create a new OpenID Connect application. The option to create or add an OpenID Connect application may vary depending on your identity provider.
  2. Fill the following fields as described. They may appear in a different order depending on your identity provider:
    • Response Type: Code
    • Grant Type: Authorization Code
    • Grants/Claims: Email
    • Redirect URL/URI: Use the subdomain from step 1
  3. The following fields may be required by your provider. If they are required, fill them as follows:
    • Token Endpoint Authentication Method: Client Secret Basic
    • Start SSO URL/Initiate Login URL: https://<subdomain>.solace.cloud. To request a subdomain from , see Request a subdomain.
  4. You need a Client Secret for the token endpoint along with the Client ID. If your identity providers does not automatically create the client secret, create it manually.
  5. Record the following information, which you need to configure your SSO settings in :
    • Client ID
    • Client Secret
    • OpenID Connect (OIDC) Discovery URL

Once your OIDC-based identity provider is set up, you can Enable SSO for your account.
