Creating and Managing Environments

  • Environments are fully supported only for 2.0.

  • is beginning the introduction of environments to in phases. Due the phased introduction, not all environment features are currently available to .

Environments in represent the operational environments within your organization. For example, you may want to have multiple development environments, a testing environment, a staging environment, and a production environment. You should be aware of these considerations for using environments:

  • You create environments at the account level.

  • You must have the Administrator role to create, modify, set a default, or delete environments.

  • Each account must have at one default environment and can have up to 50 environments.

  • Each environment must have a unique name.

  • has a default environment. You can update and use the default environment or you can delete it after you set another environment to be the default.

  • For each environment, you can turn on runtime configuration to enable to push configuration details in applications to the operational Solace event brokers represented in the environment. For more information about setting up runtime configuration, see Configuring Event Brokers from Setting Queue Configurations in [%=Product-Names.eventportal_product_short%].

You can perform the following task for environments:

  • Creating an Environment
  • Setting an Environment As Default
  • Updating an Environment
  • Setting User Access to an Environment
  • Deleting an Environment

Creating an Environment

You must be an Administrator to create environments. To create an environment, perform these steps:

  1. Log in to the . The URL to access the Cloud Console differs based on your authentication scheme. For more information, see Login URLs for [%=Product-Names.cloud_product_long%] Console.
  2. On the navigation bar, select User & Account > Environments.
  3. Click Create Environment.
  4. In the Create Environment dialog box, enter an Environment name.
  5. (Optional) Enter a Description for the environment.
  6. Click Create.

Setting an Environment As Default

provides a default environment for every organization. If you don't specify an environment during creation, the new service is added to the default environment. Users with the administrator role can set a different environment as the default. Users with the administrator role can also delete the original default environment provided by after making a different environment the default.

  1. On the navigation bar, select User & Account > Environments.
  2. In the list of environments, click More Actions for the environment that you want to set as the default and select Set As Default.

    The Default indicator appears beside the name of the environment.

Updating an Environment

Users with the Administrator role can change the name or description of any environment. For more information about adding and updating modeled event meshes in an environment, see Modeled Event Meshes.

Users with the Administrator role can change the name or description of any environment. They can also enable runtime configuration for environments to allow the configuration of Solace event brokers from . For more information about setting up runtime configuration, see Configuring Event Brokers from Setting Queue Configurations in [%=Product-Names.eventportal_product_short%].

For more information about adding and updating modeled event meshes in an environment, see Designing Modeled Event Meshes.

To update an environment, perform these steps:

  1. On the navigation bar, select User & Account > Environments.
  2. In the list of environments, click More Actions for the environment that you want to update and select Edit.
  3. Update the and description as necessary, then click Save.
  4. To allow application configuration details to be pushed to runtime s connected to the environment:
    1. In the list of environments, click More Actions for the environment you want to update, and select Enable Runtime Configuration.
    2. Click Enable.
  5. To stop allowing application configuration details to be pushed to runtime s connected to the environment:
    1. In the list of environments, click More Actions for the environment you want to update, and select Disable Runtime Configuration.
    2. Click Disable.

Setting User Access to Environments

The level of access you have to the resources in an environment depends on the role you're assigned in and the permissions granted to that role. Roles can be assigned directly to users or, if your organization has SSO enabled, through user groups. For more information, see Managing Users, Groups, Roles, and Permissions.

Users with the Administrator and roles have access to all modeled event meshes in all environments in [%=Product-Names.ep_rt_event_manager%]. Users with the role have limited viewing and editing capabilities. They can't view a modeled event mesh unless they are granted access to the environment containing the modeled event mesh by an Administrator or .

The following access levels can be granted to s:

s with Viewer access to an environment can view the objects in modeled event meshes that are part of the environment in . To open objects in the modeled event mesh, users must also have access for the application domain that the objects are contained in.
s with Editor access to an environment can perform the same functions as users with Viewer access. They can also perform these functions:
  • add and remove applications in a modeled event mesh if they also have at least Editor level access to the application domain containing the applications
  • rearrange the graph layout for the modeled event mesh

For more information about the permissions associated with the Viewer and Editor access levels, see Configuring User Access to [%=Product-Names.eventportal_product_short%].

Providing User Access to an Environment

Administrator or s can give s access to environments. To give s access, perform these steps:

  1. On the navigation bar, select User & Account > Environments.
  2. In the list of environments, click More Actions for the environment that you want to update and select Set User Access.
  3. Perform one of the following actions:

    • If your organization doesn't have user groups enabled, click Add User.

    • If your organization has user groups enabled, click Add and select either Add User Group or Add User.

  4. Click Add User.

  5. In the Name list, type or select the name of the user or user group you want to give access to. Only users and user groups with the role whom you have not already set an access level for appear in the list.
  6. In the Access Level list, select either Viewer or Editor to give access to the user or user group.
  7. Repeat steps 3-5 to give access to another user or user group.
  8. To remove access, click Remove in the row for the user or user group.
  9. Click Save.

Deleting an Environment

Users with the Administrator role can delete environments if they do not contain datacenters or event broker services and they are not the default environment. When you delete an environment, you also delete all of the modeled event meshes in the environment. However, application versions and associated objects in the modeled event mesh are not deleted from their application domains in .

To delete an environment, perform these steps:

  1. On the navigation bar, select User & Account > Environments.
  2. In the list of environments, click More Actionsfor the environment that you want to delete and select Delete.
  3. Confirm that you want to delete the environment and all of the modeled event meshes it contains.
  4. Click Delete.
