Configuring DMR for an Event Mesh via the

We recommend that you configure your event mesh using when possible. In some cases, you can configure an event mesh using for these reasons:

  • You are connecting to on-premises  event brokers (e.g., Solace  s or Solace  s)
  • Your configuration is custom or you have a DMR configuration that is specific to your organization.

Here, we show you how to use to configure an event mesh with three s named My_First_Service, My_Second_Service, and My_Third_Service:

  2. On the navigation bar, select .
  3. On the Services page, click on the card for the and then click Open . For event brokers in a private network, the connection must be initiated locally.

  4. In , select Message VPN. Note that the Message VPN must be the same as the one in the existing event mesh.

  5. In , select Clustering on the navigation bar.

  6. On the Clustering page for your service, select the External Links tab, and then click Click to Connect.

  7. On the New External Link page, in the Select Link Destination step, beneath Where to connect to, select Cloud.

  8. In the Cloud Credentials section, enter the account credentials for in one of following ways:
    • Select Token and paste the API token in the Token field. For information about creating an API token, see Managing API Tokens.
    • Select Username & Password and then your cloud credentials in the Username and Password fields.
  9. Click Configure DMR Bridge at the bottom. Depending on the broker version and if this is the first DMR external link you're creating, the button might be Select Workspace and then for subsequent links it is Configure DMR Bridge.
  10. In the Configure DMR Bridge step, select the name of your Message VPN for the service from the Local Message VPN drop-down list.

  11. In the Remote Message Service drop-down list, multiple service names are available (including the service name your connected to). Choose the name of another service that you added to visual event mesh map, and then click Remote Connection Setup. In our example, we selected My_Second_Service.

  12. In the Remote Message Service Connection pane, select Local, set the Authentication Scheme to Basic. The other Remote and Local Cluster fields are pre-filled for you with the cluster passwords.
  13. In the Transport section, enable the TLS enabled toggle if necessary.
  14. Click Apply or Create Link and Test Connection.

  15. After the link is configured (this normally takes a few minutes), you'll see the result. Click Exit and then repeat the steps in this section to configure the links between other s that you want to be part of your event mesh. Repeating the steps is necessary to configure the links between another service when you have more than two s to connect. If you only have two s, the link that's created is bidirectional and no further configuration is required.

    You have now configured the necessary DMR external links for your event mesh.
