Message Spool Expansion Usage

If you have the message spool expansion capability, you can configure message spool size of your during service creation, and scale-up the message spool size of your s while in-service. Each limit of message spool expansion provides 500 gigabytes (GB) of spool size. You use the GB as you configure and scale-up message spool size beyond the default class size for your s.
