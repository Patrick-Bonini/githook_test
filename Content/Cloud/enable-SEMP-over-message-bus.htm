Enabling SEMP Over the Message Bus

You can configure your to allow client applications to make SEMP requests over the Message Bus using defined topics. SEMP Request Over Message Bus allows clients to run Show User EXEC commands on the Message VPN for your event broker service.

To enable SEMP over the Message Bus:

  2. Select from the navigation bar.
  3. On the Services page, click the for which you want to enable SEMP over the Message Bus.
  4. On the service details page, select the Manage tab and then click Advanced Options.
  5. Next to Solace Element Management Protocol (SEMP) over Message Bus, toggle to enable or disable the feature.

Enabling SEMP Over the Message Bus only allows Show commands on the specified Message VPN.

By default, the SEMP Request Over Message Bus service is disabled for enhanced security and to keep your SEMP Show commands hidden.

You might want to enable SEMP over the Message Bus to give other teams (for example, your application development teams) access to SEMP Show commands without providing admin access. By doing this, messaging applications that use APIs can only view your settings without changing them.

Another use case might be to allow access to SEMP Show commands to a new connector that you're connecting to your event broker service. This connector will then be able to read the backlog for the queues that it connects to.
