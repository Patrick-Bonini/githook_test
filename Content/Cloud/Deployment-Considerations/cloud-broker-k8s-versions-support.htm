Supported Kubernetes Versions

In deployments for s, you (the customer) manage the Kubernetes environment. It's important that you to understand the Kubernetes version and its compatibility with the versions that are used in . Depending on the version of the Kubernetes, a minimum version of the is required.

If you are upgrading your Kubernetes cluster, recommends that you first upgrade your s to ensure compatibility and minimize outage time. For information about upgrading your s, see Software Event Broker Versions and Support in [%=Product-Names.cloud_product_short%].

In general, most versions of are supported on the listed versions of cloud and on-premises deployments of Kubernetes. If a version of Kubernetes is not listed, it is not supported at this time. If there are newer versions of Kubernetes that you require, contact for more information.

  • A minimum broker version is listed where required, otherwise support on the specified version of the Kubernetes provider or distribution is indicated using the following symbols:
    • —A greenblue check mark shows full support based on a minimum broker version, if listed.
    • —A gray check mark shows that the broker version has full support but has not been fully tested on the specific Kubernetes release at this time. In addition, there may be known issues or additional work required to deploy s on the Kubernetes cluster. Contact for more information.
    • Note that if a version for a Kubernetes provider or distribution isn't listed in the table below, there are several reasons that can include:
      • The provider or distribution never supported the version (for example, some providers only support the even version numbers of Kubernetes).
      • The Kubernetes version has fallen out of support [or reached end-of-life (EOL)] as specified by the provider (see the website for the provider or distribution).
