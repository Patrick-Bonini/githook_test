Using AWS Transit Gateway for EKS Connectivity

Use of AWS Transit Gateway for EKS connectivity is a Controlled-Availability (CA) feature. Contact to see if this feature is suitable for your use case and deployment requirements.

Customers deploying to an Amazon Elastic Kubernetes Service (EKS) cluster can use AWS Transit Gateway, as a controlled availability feature, to establish connectivity.

only supports using this feature for attaching a to an AWS Transit Gateway you (the customer) manage. We do not support hosting the AWS Transit Gateway ourselves.

In addition to using VPC Peering, customers deploying to an AWS Elastic Kubernetes Service (EKS) cluster in a can use AWS Transit Gateway as a controlled availability feature to establish connectivity. As long as the customer can establish IP (Layer 3) connectivity between the VPCs, it will be sufficient for connectivity between client applications and event broker services .

AWS Transit Gateway offers both benefits and limitations you should be aware of. See Transit gateways and Transit gateway attachments to a VPC in the AWS documentation for more information about how using Transit Gateway affects your EKS cluster.
