In s, we deploy the and s in a region controlled by . If you require private between the s in your and the virtual private cloud (VPC) where your client applications reside, we offer several networking options supported by your chosen cloud provider.

When deciding on the networking options for your , consider the requirements of your entire estate, including:

  • whether you need bidirectional or unidirectional network connectivity

  • the amount of s you want, or may require in the future

  • bandwidth limitations

  • support for multiple regions

  • support for transitive routing

Virtual Private Cloud (VPC) and Virtual Network (VNet) Peering

With virtual private cloud (VPC) or virtual network (VNet) peering, connects your region with another in the same cloud provider, or with your network. Multiple regions and networks can be connected together this way.

The diagram below provides examples of VPC/VNet peering between two s and VPC/VNet peering between a and a customer network.

 

VPC/VNet peering is available for every cloud provider deploys s to:

  • Azure VNET Peering

  • Amazon VPC peering

  • Google VPC peering

Benefits
  • VPC/VNet peering is the simplest network solution for .
Security Considerations

  • This can be useful if you want to enable certain features, like REST Delivery Points for your event broker services, which require both outbound and inbound connections.

Drawbacks

  • Transitive peering is only available for s deployed to Azure Kubernetes Service (AKS).

Site-to-Site Virtual Private Network (VPN)

With site-to-site virtual private networking (VPN), connects the with another , or with your network. Multiple s and networks can be connected together this way.

Using site-to-site VPN for the for your is an additional add-on purchase. Contact Solace for information.

This diagram below provides examples of site-to-site VPN network between two s deployed to different cloud providers, and site-to-site VPN between a and a customer network.

 

Site-to-site VPN is available for every cloud provider deploys s to:

  • Azure site-to-site VPN

  • Amazon site-to-site VPN

  • Google site-to-site VPN

If you choose to use site-to-site VPN, you must complete the site-to-site VPN questionnaire for deployments. For more information, see Questionnaire: Site-to-Site VPN for [%=Product-Names.dedicated_region%] Deployments .

Benefits
  • Site-to-site VPN is the only option if your VPC or network is partially or completely on-premises, or if you want to connect multiple s in different cloud providers (multi-cloud).
Drawbacks
  • Site-to-site VPN is an add-on purchase, and is not included by default for deployments.
