Okta with

You can integrate Okta with to enable single sign-on (SSO) and multi-factor authentication (MFA) for your enterprise accounts.

After you have configured Okta, your organization will receive a customized URL to log in to . If your users have already signed in to their Okta account, they will automatically be signed in to . Otherwise, they will be directed to your organization's Okta sign-in page to enter their details. Your organization's Okta settings determine whether multi-factor authentication (MFA) is used.

If a user belongs to multiple accounts that are associated with your organization's Okta ID, they can switch between those accounts without logging in to the accounts separately.

Register as an Okta Application

When you set up as an application in Okta, a Client ID, Client Secret, and Discovery URL is generated that will be used to bind your account. These information allow your organization's users to be authenticated by Okta.

To register as an application in Okta, request a preferred subdomain and perform the following steps:

  1. In the Okta's admin dashboard, click the Applications tab and select Applications.

  2. Click the Add Application button.

  3. Click the Create New App button.

  4. On the Create a New Application Integration dialog, do the following:
    1. For Platform, select Web from the dropdown list.
    2. In the Sign on method options, select OpenID Connect.
    3. Click Create.

  5. On the Create OpenID Connection Integration page, do the following:
    1. Write your Application name as PubSub+ Cloud.
    2. In the Login redirect URIs field, enter the URL provided by (for example: https://<your-subdomain>.solace.cloud/sso/login). See Request a Preferred Subdomain to learn how to get the URL. Note that URLs can differ depending on the region you are in, see Login URLs for the [%=Product-Names.console_ui_long%] for more information.

  6. Click Save. The application will be created and Active.

  7. Select the General tab, and take note of the following that will be required for your SSO Configuration in :
    • Client ID
    • Client Secret

  8. Make a note of the following information, which you will need when configuring your SSO in :

    • Client ID
    • Client Secret

    • Discovery URL
