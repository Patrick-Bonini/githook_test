Configuring Kafka Bridging

Kafka bridging allows you to configure message flow in both directions between a Kafka cluster and a   software event broker or an in . You configure the Kafka bridge within the event broker and no external Kafka Connect infrastructure is required to pass messages to and from Kafka.

To configure Kafka bridging, you need to configure a Kafka sender and a Kafka receiver.

  • A Kafka receiver receives events from one or more Kafka topics, converts the events to Solace Message Format (SMF), and publishes them to topics on the event broker.

  • A Kafka sender takes SMF messages from one or more queues, converts the messages to Kafka events, and sends them to Kafka topics on a remote Kafka cluster.

For more information about how Kafka bridging works, see Kafka Bridging Overview.
