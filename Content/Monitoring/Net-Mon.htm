Network Monitoring

The event broker provides various mechanisms to monitor its status and troubleshoot network problems by displaying, for example, summary screens of network statistics. There are a number of CLI commands that can be run to collect this data and investigate network level issues. They classify as follows:

  • Display IP Networking status
  • Display event broker status
  • Monitor event broker status
  • Display subscription database contents
  • Display network statistics

Each CLI command also has an equivalent SEMP counterpart; hence, a monitoring application can obtain this information using request/reply SEMP as described in Request / Reply Monitoring using SEMP.

Refer to the section “Network Monitoring” in the [SolAppMaint] guide for more information.
