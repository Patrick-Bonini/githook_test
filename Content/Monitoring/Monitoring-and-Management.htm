Configuring Monitoring

Solace PubSub+ event brokers can be configured to generate events that provide management and status information. These events are sent to the system logs and can also be published as direct messages to well-known topics via the event broker message bus. For details, see the following sections:

  • Configuring Events and Thresholds—customize the format of generated events and their output destinations, and configure the threshold values required for some types of events.
  • Displaying and Clearing Logs—display log entries and clearing entries from logs
  • Last Restart Reason—obtain information about the last time the broker restarted, including the type of shutdown, and the reason for shutting down
  • Monitoring Events Using Syslog— use the system logging mechanism to monitor the PubSub+ event broker
  • Configuring the Log File Retention Policy— configure the broker to retain logs for a specified number of days
  • Monitoring Hardware Blades—monitor the state and performance of the hardware blades in a PubSub+ appliance
  • Receiving Message Bus Events—receive system event messages published over the event broker message bus
  • Software Event Broker Health Monitoring—monitor event broker performance and stability
  • Subscribing to Message Bus Events—subscribe to standard topics to receive system events
