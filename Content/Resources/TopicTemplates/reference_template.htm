Reference

Reference topics provide data that supports users as they perform a task. Reference topics might provide lists and tables that include product specifications, parts lists, and other data that is often “looked up” rather than memorized. A reference topic also can describe commands in a programming language or required tools for a series of maintenance tasks.

Reference topics provide quick access to fact-based information. In technical information, reference topics are used to list product specifications and parameters, provide essential data, and provide detailed information on subjects such as the commands in a programming language. Reference topics can contain any subject matter that has regular content, such as ingredients for food in recipes, bibliographic lists, catalog items, and so on.

Reference topics contain the following elements:

  • Title—The name of the reference object or property
  • A short (less than one sentence) description of the item (no title)
  • Syntax or signature content (for example, a command-line utility's calling syntax or an API's signature). Contains a brief, possibly diagrammatic description of the subject's interface or high-level structure. Includes an appropriate title ("Syntax", "Signature", "Format", etc.)
  • Description—A detailed explanation of the reference object or property. May contain subsections, but long descriptions may be better as separate Concept topics.
  • Example (optional)—Provides example(s) that illustrate or support the current topic.
  • Properties (optional)— Lists properties or attributes of a subject and their types, values, and descriptions. May be presented in a table that organizes information into rows and columns.
