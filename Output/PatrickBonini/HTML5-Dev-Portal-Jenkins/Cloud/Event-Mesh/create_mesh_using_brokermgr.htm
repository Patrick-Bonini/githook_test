Creating an Event Mesh using PubSub+ Broker Manager (Legacy)

We recommend that you use Mesh Manager to create your event meshes when possible.

If you have created your event mesh using Mesh Manager, Solace recommends that you do not modify the mesh using PubSub+ Broker Manager. If you require advanced settings, contact Solace.

In some cases, your configuration may require that you configure your event mesh using Broker Manager because of configuration requirements.

At this time, you cannot create a visualization of a map prior to configuring the map in Mesh Manager. After you've decided on the event broker services that are to part of your event mesh, and configured your event mesh, you can use Mesh Manager to scan for your new mesh and the visualization of the event mesh is available.

To start, can use Broker Manager to configure DMR external links between your event broker services.

For more information, see Configuring DMR for an Event Mesh via the Broker Manager.

Configuring DMR for an Event Mesh via the Broker Manager

We recommend that you configure your event mesh using Mesh Manager when possible. In some cases, you can configure an event mesh using Broker Manager for these reasons:

  • You are connecting to on-premises PubSub+ event brokers (e.g.,  software event brokers or  appliances)
  • Your configuration is custom or you have a DMR configuration that is specific to your organization.

Here, we show you how to use Broker Manager to configure an event mesh with three event broker services named My_First_Service, My_Second_Service, and My_Third_Service:

  1. Log in to the PubSub+ Cloud Console if you have not done so yet. The URL to access the Cloud Console differs based on your authentication scheme. For more information, see Logging In to the PubSub+ Cloud Console.

  2. On the navigation bar, select Cluster Manager .
  3. On the Services page, click on the card for the event broker service and then click Open PubSub+ Broker Manager. For event brokers in a private network, the connection must be initiated locally.

  4. In Broker Manager, select Message VPN.

  5. In Broker Manager, select Clustering on the navigation bar.

  6. On the Clustering page for your service, select the External Links tab, and then click Click to Connect.

  7. On the New External Link page, in the Select Link Destination step, beneath Where to connect to, select Solace Cloud.

  8. In the Solace Cloud Credentials section, enter the account credentials for PubSub+ Cloud in one of following ways:
    • Select Token and paste the API token in the Token field. For information about creating an API token, see Managing API Tokens.
    • Select Username & Password and then your cloud credentials in the Username and Password fields.
  9. Click Configure DMR Bridge at the bottom. Depending on the broker version and if this is the first DMR external link you're creating, the button might be Select Workspace and then for subsequent links it is Configure DMR Bridge.
  10. In the Configure DMR Bridge step, select the name of your Message VPN for the service from the Local Message VPN drop-down list.

  11. In the Remote Message Service drop-down list, multiple service names are available (including the service name your connected to). Choose the name of another service that you added to visual event mesh map, and then click Remote Connection Setup. In our example, we selected My_Second_Service.

  12. In the Remote Message Service Connection pane, select Local, set the Authentication Scheme to Basic. The other Remote and Local Cluster fields are pre-filled for you with the cluster passwords.
  13. In the Transport section, enable the TLS enabled toggle if necessary.
  14. Click Apply or Create Link and Test Connection.

  15. After the link is configured (this normally takes a few minutes), you'll see the result. Click Exit and then repeat the steps in this section to configure the links between other event broker services that you want to be part of your event mesh. Repeating the steps is necessary to configure the links between another service when you have more than two event broker services to connect. If you only have two event broker services, the link that's created is bidirectional and no further configuration is required.

    You have now configured the necessary DMR external links for your event mesh. To see it in action, use the Try Me! tab in PubSub+ Cloud to create a subscription in one event broker service, publish from another event broker service and see your message dynamically forward from one event broker services to the other. To verify your link, in Broker Manager, select Clustering from the navigation bar.
